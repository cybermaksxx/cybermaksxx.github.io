Complete Nmap Guide

Master network reconnaissance, stealth scanning, and firewall evasion techniques

Tool: Nmap (Network Mapper)

Category: Network Scanning & Enumeration

Difficulty: Beginner to Advanced

Website: https://nmap.org

Introduction to Nmap

One of the most underestimated phases in penetration testing is reconnaissance and enumeration. Many newcomers, often due to inexperience, undervalue its critical importance, rushing toward exploitation without a solid foundation. A thorough enumeration phase is what separates a successful, targeted attack from a noisy, scattergun approach.

At the heart of this initial phase lies Nmap (Network Mapper), the quintessential tool for network discovery and security auditing. Its story began in 1997 when a young hacker named Gordon "Fyodor" Lyon created it to map his own network and understand the traffic flowing through it. He published it in an article for Phrack Magazine, and what started as a simple port scanner evolved into a community-driven, open-source project that defined the standard for network exploration.

💡 Professional Insight: While modern alternatives like RustScan have emerged, Nmap remains the industry standard due to its extensive capabilities, scripting engine, and proven reliability in real-world scenarios.

Installation

First things first... If you are running Kali or Parrot Linux, you have this tool installed by default. However, if you don't, here's how you can install it:

# Ubuntu/Debian
sudo apt update && sudo apt install nmap

# CentOS/RHEL
sudo yum install nmap

# Windows
# Download from https://nmap.org/download.html

# macOS
brew install nmap
📝 Note: For advanced scanning techniques like SYN scans, you'll need root privileges on Unix-based systems.

Basic Syntax

Every Nmap operation follows a consistent and logical structure that forms the foundation of network reconnaissance. Understanding this syntax is crucial for effective scanning.

nmap [scan type] [options] [target]

Syntax Components:

  • nmap - The command that invokes the tool
  • [scan type] - Defines the scanning technique (e.g., -sS, -sT)
  • [options] - Customizes and enhances your scan
  • [target] - The objective (IP, range, or subnet)
🚀 Pro Tip: When no scan type is specified and you have root privileges, Nmap automatically uses the TCP SYN Stealth Scan (-sS) - your default workhorse for efficient reconnaissance.

Scan Types

Nmap offers a diverse arsenal of scanning techniques, each designed for specific scenarios and objectives. Mastering these is essential for effective ethical hacking.

TCP

TCP Connect Scan (-sT)

The TCP Connect scan performs a complete three-way handshake with every target port, making it the most reliable but also the most detectable scanning method.

nmap -sT [target]

How It Works

Nmap completes the full TCP handshake (SYN → SYN-ACK → ACK) for each port, then immediately closes the connection. This method uses the operating system's native connect() function.

✅ Advantages

  • Maximum reliability and accuracy
  • No root privileges required
  • Clear, unambiguous results
  • Works through most firewalls

❌ Disadvantages

  • Loud and easily detected
  • Slower than stealth methods
  • Creates log entries on target
  • Can be blocked by stateful firewalls

SYN

SYN Scan (-sS) - The Stealth Workhorse

The SYN scan, also known as "half-open" scanning, is the default and most popular scanning method when run with root privileges. It offers the perfect balance of speed, stealth, and reliability.

sudo nmap -sS [target]

Stealth Mechanics

Instead of completing the three-way handshake, SYN scans send a RST packet after receiving the SYN-ACK, preventing the connection from being fully established and reducing detection.

✅ Advantages

  • Fast and efficient
  • Stealthy - avoids application logging
  • Bypasses legacy IDS systems
  • Industry standard for professionals

❌ Disadvantages

  • Requires root/administrator privileges
  • Can crash unstable services
  • Detected by modern security systems
  • Some firewalls block SYN packets

UDP

UDP Scan (-sU)

UDP scanning is essential for discovering services that don't use TCP, but it's significantly slower and more challenging due to UDP's connectionless nature.

sudo nmap -sU [target]

Critical UDP Services

  • DNS (53) - Domain Name System
  • SNMP (161) - Network Management
  • DHCP (67/68) - Dynamic Host Configuration
  • TFTP (69) - Trivial File Transfer
# Efficient UDP scanning
nmap -sU --top-ports 100 [target]
nmap -sU -p 53,161,162,69 [target]
⚠️ Important: Full UDP scans can take hours or even days. Always use --top-ports or specify critical ports to scan efficiently.

Firewall Evasion Techniques

Modern networks are protected by various firewall technologies. Understanding how to bypass them is crucial for comprehensive security assessment.

Advanced Evasion Methods

Packet Fragmentation (-f)

nmap -f [target]

Breaks packets into smaller fragments to bypass simple firewall rules and IDS systems.

Decoy Scanning (-D)

nmap -D decoy1,decoy2,decoy3,ME [target]

Uses decoy IP addresses to hide your real source IP in log files.

Source Port Manipulation

nmap --source-port 53 [target]
nmap -g 80 [target]

Spoofs source port to appear as legitimate traffic (DNS, HTTP, etc.).

Timing and Performance

nmap -T0 [target] # Paranoid (very slow)
nmap -T1 [target] # Sneaky
nmap -T2 [target] # Polite
nmap -T3 [target] # Normal
nmap -T4 [target] # Aggressive
nmap -T5 [target] # Insane

Same-Network Scanning Strategies

When operating within the same network segment, you gain significant advantages for reconnaissance and enumeration.

Local Network Advantages

  • ARP-based host discovery
  • Reduced firewall restrictions
  • Faster scan times
  • Direct access to internal services

ARP Scan (-PR)

nmap -PR 192.168.1.0/24
nmap -sn 192.168.1.0/24 # Host discovery only

Internal Service Enumeration

# Common internal services
nmap -p 22,23,53,80,443,445,3389 [target]

# SMB/CIFS shares
nmap -p 445 --script smb-enum-shares [target]

# RDP services
nmap -p 3389 --script rdp-enum-encryption [target]

Nmap Scripting Engine (NSE)

The NSE transforms Nmap from a simple port scanner into a powerful vulnerability assessment and exploitation tool.

# Script categories
nmap --script safe [target]
nmap --script vuln [target]
nmap --script exploit [target]
nmap --script auth [target]

# Specific scripts
nmap --script http-enum [target]
nmap --script ssl-enum-ciphers [target]
nmap --script smb-vuln-ms17-010 [target]
🔧 Pro Tip: Always start with 'safe' scripts during initial reconnaissance and escalate to more intrusive scripts as you gain authorization and understanding of the target environment.

Practical Scanning Examples

Real-world scanning scenarios for different objectives and environments.

# Quick network discovery
nmap -sn 192.168.1.0/24

# Comprehensive scan
nmap -A -T4 [target]

# Stealthy enumeration
nmap -sS -sV -O -T4 [target]

# Vulnerability assessment
nmap -sV --script vuln [target]

# Full port scan
nmap -p- -T4 [target]

Mastering Nmap: The Ethical Hacker's Journey

Nmap is more than just a port scanner—it's a comprehensive reconnaissance platform that forms the foundation of professional security assessment. From basic network mapping to advanced firewall evasion and vulnerability detection, Nmap provides the tools needed for thorough, professional-grade security testing.

Key Takeaways for Ethical Hackers:

  • Always begin with proper authorization and scope definition
  • Start with stealthy techniques and escalate as needed
  • Understand the trade-offs between speed, stealth, and reliability
  • Use NSE scripts responsibly and understand their impact
  • Document your findings thoroughly for professional reporting

Remember: With great power comes great responsibility. Use these skills to strengthen defenses, conduct authorized testing, and contribute to a more secure digital world.
